## Technical SEO Findings

### 🔴 P1 — robots.txt Blocked by DataDome
- **Issue:** `curl https://www.motorinnautogroup.com/robots.txt` returns DataDome challenge HTML (200 status but HTML content, not `text/plain`). Non-browser requests (including some Googlebot passes) receive the JS challenge page.
- **Impact:** HIGH — If verified Googlebot hits this and can't parse robots.txt, it defaults to "crawl everything." More dangerous: if DealerOn's new domain also has DataDome misconfigured, launch-day crawls could behave unpredictably.
- **Evidence:** `curl robots.txt` → returns `<html>... Please enable JS and disable any ad blocker ...` (DataDome challenge)
- **Fix:** In DataDome admin, add Googlebot, Bingbot, and other verified search crawlers to the bypass allowlist. DealerOn should handle this — confirm it's configured pre-launch.
- **Priority:** P1 — resolve before/immediately after April 10 migration

### 🔴 P1 — DealerOn Migration Tomorrow (April 10)
- **Issue:** Full platform migration from Carbase to DealerOn. Risk of redirect chain failures, canonical drift, and indexation drops.
- **Impact:** HIGH — if redirects aren't 1:1 (301), pages currently indexed could get deindexed and lose ranking history.
- **Evidence:** Context file notes "DealerOn migration ~April 10, 2026." Sitemap shows active inventory URLs (priority 0.75, updated today).
- **Fix:**
  1. Immediately after launch: verify 10-15 high-priority URLs redirect 301 → new URLs (no chains)
  2. Check new sitemap exists and submit to GSC (GSC connection planned April 11)
  3. Confirm canonical tags on new site point to correct HTTPS www URLs
  4. Monitor GA4 for any traffic cliff in the 48 hours post-launch
- **Priority:** P1 — active watch item starting April 10

## Schema Markup Findings

### 🟡 P2 — HTTP Instead of HTTPS in Schema Context
- **Issue:** All 3 schema blocks use `"@context": "http://schema.org"` instead of `"@context": "https://schema.org"`
- **Impact:** MEDIUM — Google still processes http:// schema, but https:// is the current standard and may affect rich result eligibility
- **Evidence:** Live schema extracted via browser: `{"@context":"http://schema.org"...}` across WebSite, AutomotiveBusiness, AutoDealer schemas
- **Fix:** Update all JSON-LD blocks to use `https://schema.org`
- **Priority:** P2 — fix in DealerOn template config

### 🟡 P2 — WebSite Schema Name Has Trailing Slash
- **Issue:** `"name": "Motor Inn Auto Group/"` — trailing slash is a data error
- **Impact:** LOW-MEDIUM — could surface incorrectly in knowledge panel / rich results
- **Evidence:** Extracted schema: `"name":"Motor Inn Auto Group/"`
- **Fix:** Remove trailing slash — should be `"name": "Motor Inn Auto Group"`
- **Priority:** P2

### 🟡 P2 — Phone Number Inconsistency in Schema
- **Issue:** Two phone numbers in schema: AutomotiveBusiness lists `+1 (712) 362-5834`, AutoDealer (Carroll) lists `(712) 522-2526`. Context file shows (712) 522-2526 as the canonical number.
- **Impact:** MEDIUM — inconsistent NAP (Name/Address/Phone) across schema blocks hurts local SEO trust signals
- **Evidence:** Extracted schema comparison
- **Fix:** Standardize to `(712) 522-2526` in E.164 format (`+17125222526`) across all schema. Remove or verify the 362-5834 number.
- **Priority:** P2

### 🟡 P2 — Missing Review/AggregateRating Schema
- **Issue:** Site has a `/reviews` and `/testimonials` page in the sitemap, but homepage schema has no `aggregateRating` property on AutoDealer
- **Impact:** MEDIUM — star ratings in SERPs require AggregateRating schema; competitor dealers using this get visual advantage in results
- **Evidence:** Schema extracted from homepage — no AggregateRating node present
- **Fix:** Add `aggregateRating` to the AutoDealer schema block with ratingValue, reviewCount from actual reviews
- **Priority:** P2

### 🟢 P3 — No FAQPage Schema on FAQ Content Pages
- **Issue:** Site has `/credit-score-qanda`, `/car-warranties-qanda`, `/faqs-vehicle-service` in sitemap — none likely have FAQPage schema
- **Impact:** LOW — missed rich result opportunity (FAQ accordion in SERPs)
- **Fix:** Add FAQPage schema to each Q&A page
- **Priority:** P3 — address post-migration

## On-Page SEO Findings

### ✅ Homepage Title — GOOD
- `Toyota & Chevrolet Dealer in Carroll, IA | Motor Inn Auto Group`
- Primary keyword near front, brand at end, appropriate length (~60 chars)

### ✅ Homepage Meta Description — GOOD
- `Motor Inn Auto Group is a Toyota and Chevrolet dealership selling new & used cars, trucks, vans, crossovers, and SUVs. Visit us today at our dealership in Carroll, Iowa.`
- Includes keywords, has a CTA, appropriate length (~168 chars — slightly long but acceptable)

### ✅ Canonical Tag — GOOD
- `<link rel="canonical" href="https://www.motorinnautogroup.com/">` — self-referencing, correct

### 🟡 P2 — Weak Heading Structure on Homepage
- **Issue:** Only 3 H2s: "FIND YOUR NEXT VEHICLE," "SHOP BY BRAND," "TESTIMONIALS." No location/trust/service H2s.
- **Impact:** MEDIUM — Google uses headings to understand page topical relevance. Missing signals for service area, trust, and brand depth.
- **Fix:** DealerOn template should add H2s for service area ("Serving Carroll, Spirit Lake & Surrounding Iowa Communities"), about/trust section, and service/financing CTAs
- **Priority:** P2

### 🟡 P2 — Mixed-Case Blog URLs
- **Issue:** Blog/content articles have inconsistent URL casing: `/SUV-For-Every-Stage-Of-Life-IA`, `/Why-Buy-a-Toyota-Crossover-Carroll-IA`, `/2019-Chevrolet-Equinox-VS-2019-Ford-Escape-IA` — mixed uppercase
- **Impact:** MEDIUM — Google treats URLs as case-sensitive. Inconsistency can cause duplicate indexation and link equity splits.
- **Fix:** DealerOn migration should force all blog URLs to lowercase with 301s from old mixed-case URLs
- **Priority:** P2

### 🟢 P3 — Blog Content Heavily Outdated
- **Issue:** Most blog articles last modified 2024-05-09. Several are 2020/2021 model year pages (2021 RAV4, 2020 Tundra, etc.) still live and indexed.
- **Impact:** LOW-MEDIUM — outdated content signals stale E-E-A-T; 2020/2021 model year pages waste crawl budget
- **Fix:** After DealerOn launch — audit stale pages, either update or noindex/redirect outdated model year pages to current inventory
- **Priority:** P3
